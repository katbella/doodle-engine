title: Assets in Studio

Studio can import media from anywhere on your computer and place a game-owned copy in the correct project folder.

## Add an asset

Open the character, location, item, map, interlude, dialogue node, or game setting that should use the asset. Select **Choose file…** beside the relevant field, then choose the file from your computer.

[![A location asset field with its file picker](/images/studio/asset-fields.png)](/images/studio/asset-fields.png)

Studio copies the file into your game and fills in the field. Your original file remains where it was.

If the game already contains a different file with the same name, Studio gives the imported copy a unique name.

## Where assets appear

| Content | Available media |
| --- | --- |
| Character | Portrait |
| Location | Banner, music, and ambient sound |
| Item | Image or icon |
| Map | Map image |
| Interlude | Background, banner, music, voice, and sound effects |
| Dialogue node | Voice and an optional portrait override |
| Game shell | Interface images, music, and sounds |

Use **Preview** to see and hear assets in the game’s renderer. Studio’s dialogue playtester focuses on text, choices, and game state; it does not play voice lines or display portraits.

See [Assets & Media](/guides/assets-and-media/) for supported formats and how the game loads media.
